## 3. Changes Made

**Issue 1 — Re-fire loop (LES-000022):** Removed `issue_comment: [created]` from the workflow trigger block entirely. Added a prominent comment explaining why it was removed and the correct way to re-add it when a bot actor identity is established. Issues can be re-edited to trigger re-parsing, which is sufficient for v1 baseline.

**Issue 2 — Gates step ordering (LES-000023):** Added `if: always()` to the "Read gate check result" step in both intake and genesis jobs. Updated the label-selection logic to explicitly distinguish "FAILING_GATES was N/A or empty because the gate file was never written" (chain failed → `status:blocked`) from "FAILING_GATES was empty because all gates passed" (genuine success → `status:draft-needs-review`).

---

## 4. Why This Amendment Resolves The Lessons

LES-000022 is resolved by removing the trigger — no `issue_comment` event, no re-fire. LES-000023 is resolved by ensuring the gates step always runs regardless of whether the chain succeeded, and by making the label logic's handling of the failure case explicit rather than accidentally mapping to the success case.
