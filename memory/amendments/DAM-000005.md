## 1. Source Lesson

LES-000010 — `DAM-000001`'s fix to `issue_parser.py`'s `GITHUB_OUTPUT` writing was real but incomplete: it never added the corresponding job-level `outputs:` mapping, so the genesis chain's `brief` had never actually flowed between jobs in a live run. Found while building the `instruction` equivalent for BUILD and checking the existing pattern before reusing it.

This amendment also directly executes the binding resequencing guidance recorded in `DAM-000004` / `EMS_OPERATING_MODEL.md` Section 6: build a real Team 2 forward mission executor before further governance expansion.

## 3. Before / After

**Before:** `detect-mission` job's `outputs:` block exposed only `mission_type`, `platform_name`, `repo_url`, `issue_number`. The `execute-genesis` job referenced `needs.detect-mission.outputs.brief`, which had no corresponding mapping and therefore always resolved empty, regardless of what `issue_parser.py` itself produced.

**After:** `outputs:` block now also maps `brief` and `instruction`, both flowing correctly from the parser through to the genesis and build jobs respectively.

**Before:** No executor existed for any Team 2 forward mission. Intake and genesis could establish a baseline; nothing could build forward from one.

**After:** `run_build_chain.py` exists — confirms platform readiness (tolerating only the known, system-wide RG-008 gap, not platform-specific failures), then runs 5 NIM-gated passes mirroring OPR-000003 through OPR-000007 exactly, with TDA enforced as a genuine halt (a REJECTED/REVISION_REQUIRED verdict stops the chain, doctrine respected, not soft-warned past), and a hard script-level safety check preventing a FAIL verification from ever producing a RELEASE decision regardless of what the model itself returns.

---

## 4. Why This Amendment Resolves The Lesson

LES-000010 is resolved at the structural level the bug actually existed at — the job-level YAML mapping, not just the script — and the same mapping pattern is applied to `instruction` from the start, rather than introducing the identical incomplete fix a second time. The resequencing guidance from LES-000009/DAM-000004 is resolved by being directly executed, not just acknowledged: this is the BUILD executor that document said should be the next major effort.

---

## 6. Regression Prevention

No automated test was added for the GitHub Actions job-output mapping specifically — that layer is not unit-testable with the current `ems_engine` test setup, which tests Python logic, not GitHub Actions YAML wiring. The mitigation is procedural, recorded as Reuse Guidance in `LES-000010` itself: any future fix involving cross-job data flow must verify the job-level `outputs:` block explicitly, not just the script-level write. This is a known limitation of the current verification approach, disclosed rather than hidden.
