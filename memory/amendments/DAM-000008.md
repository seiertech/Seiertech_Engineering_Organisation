## 3. Before / After

**Before:** `COMMANDER_ONBOARDING_CHECKLIST.md` — title, every section header, and most prose referenced Commander specifically, despite the actual content (NIM key setup, target repo token, mission-firing format) being entirely generic across any platform.

**After:** `PLATFORM_ONBOARDING_CHECKLIST.md` — same useful content, genuinely generic, with Commander retained only as the worked example in Section 6, the identical pattern used throughout the rest of the doctrine (`STD-000004`, `MSN-000000`, `MSN-000001`).

**Before:** the doctrine inventory's document table listed 7 of 12 actual root-level files.

**After:** all 12 listed, each with a one-line purpose description, verified by direct filesystem listing rather than memory.

---

## 4. Why This Amendment Resolves The Lesson

LES-000015 is resolved the same way the original `PLATFORM_AGNOSTIC_AUDIT.md` resolved the persona instances — by genuinely rewriting the content to be reusable, not by adding a disclaimer on top of Commander-specific text. LES-000016 is resolved by verifying against ground truth (`ls *.md`) rather than trusting the table's own prior content, which is exactly the discipline `DAM-000007` established for artefact ID citations one level down — applied here one level up, to the index of documents itself.

## 5. Verification

- Confirmed via `grep -rl "COMMANDER_ONBOARDING_CHECKLIST"` that only one live reference existed (`IMPLEMENTATION_STATUS.md`) plus one correctly-historical mention in `EMS_VOCABULARY_AUDIT.md` (left as-is, consistent with how every other audit-doc historical reference has been treated this session) — both checked and the live one fixed
- Discovered and fixed an additional defect during this same check: the old `COMMANDER_ONBOARDING_CHECKLIST.md` file had not actually been deleted when the new file was created, leaving a stale duplicate on disk — removed via `git rm`
- Ran `ls *.md` directly against the repository root to build the corrected "Other Key Documents" table from ground truth, not from the table's own prior (incomplete) content

---

## 6. Regression Prevention

No automated check exists for "every root-level .md file is listed in the doctrine inventory" — this is the same category of limitation already disclosed for GitHub Actions job wiring (`DAM-000005`) and git mechanics (`DAM-000006`): not yet covered by the `ems_engine` automated test suite. The mitigation remains procedural: future platform-agnostic sweeps should explicitly include root-level documents in scope, and the doctrine inventory's document table should be spot-checked against `ls` periodically, not assumed current.
