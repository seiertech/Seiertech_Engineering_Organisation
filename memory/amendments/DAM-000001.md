## 3. Before / After

**Before:** `scan_repo.py`'s route detection regex only matched FastAPI-style `@app.get()`/`@app.post()`, despite a comment claiming Flask coverage. Flask's actual `@app.route()` decorator was never detected.

**After:** Both Flask (`@app.route()`, `@blueprint.route()`) and Django (`path(..., View.as_view())`, `urlpatterns =`) patterns added alongside the existing FastAPI/Express/Spring patterns.

**Before:** `check_readiness_gates.py` only recognised `SCAN_RESULT.json` and `INTAKE_RUN_LOG.md` as evidence — both brownfield-only artefacts. A greenfield (genesis) platform, which has no repo to scan by design, could never produce these files and was therefore structurally incapable of ever reaching READY status, regardless of completeness.

**After:** Gate definitions accept `GENESIS_RUN_LOG.md` as equivalent evidence for RG-001/002/Standards-Engineer-gate, and `ARCHITECTURE_DOCUMENT.md` as equivalent evidence for RG-004 (tech stack). Both origins now converge on the same honest ceiling.

**Before:** `issue_parser.py` extracted the `brief` field for GENESIS missions into its JSON output but never wrote it to `GITHUB_OUTPUT` — the mechanism GitHub Actions uses to pass values between job steps.

**After:** A `brief=` line added to the `GITHUB_OUTPUT` write block, enabling the genesis workflow job to actually receive the brief text.

---

## 4. Why This Amendment Resolves The Lesson

All three were discovered because the simulation exercise exercised real code paths against real (if local) fixture repos, rather than only synthetic test fixtures that happened to avoid the exact gaps. The fixes close each gap at its source rather than patching symptoms — the route detection now covers the frameworks actually claimed in its own comments, the gate checker is genuinely origin-agnostic rather than brownfield-only, and the data flow from parser to workflow is complete end to end.

## 6. Regression Prevention

A new permanent test, `test_genesis_origin_platform_can_reach_same_ceiling_as_brownfield`, added to `ems_engine/tests/test_core.py`. It builds a genesis-origin fixture and asserts only `RG-008` blocks it — exactly the same assertion structure as the existing brownfield positive-case test, so the two origins are now held to the same standard by an automated, permanent check rather than relying on manual re-testing.
