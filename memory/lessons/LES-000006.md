# LES-000006 — Batch-written personas silently end up thin

| Field | Value |

## Finding
8 of Team 1's 25 personas, written via a Python batch script (working around a bash heredoc failure earlier in the build), ended up missing 11 of the 19 standard sections other personas carried — averaging 68 lines versus siblings' 109-265. This sat unnoticed until an explicit founder-requested sense-check, despite the gap being large and structurally simple to detect.

## Impact
Nearly a third of Team 1's persona set was carrying meaningfully less reasoning depth than the rest, which would have produced weaker intake output for any platform without anyone knowing why.

## Reuse Guidance
When multiple personas (or any artefact class) are written in a batch — especially via a script rather than individually — run the structural completeness check immediately after, not at the next scheduled audit. The batch itself is the highest-risk moment for this class of gap.
