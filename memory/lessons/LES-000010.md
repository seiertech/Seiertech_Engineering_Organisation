# LES-000010 — GITHUB_OUTPUT writes were never actually exposed to other jobs

| Field | Value |
|---|---|
| Lesson ID | LES-000010 |
| Category | STRUCTURAL_BUG |
| Source | Building the BUILD chain executor (run_build_chain.py) |
| Date | 2026-06-30 |
| Doctrine Amendment | DAM-000005 |
| Status | AMENDED |

## Finding
`DAM-000001` (this same session, earlier) fixed `issue_parser.py` to write a `brief=` line to `GITHUB_OUTPUT` for GENESIS missions, and the fix was verified — at the script level. What was never checked: GitHub Actions requires a separate, explicit job-level `outputs:` mapping (`brief: ${{ steps.parse.outputs.brief }}`) for a value written to `GITHUB_OUTPUT` in one job to become visible to a different job via `needs.detect-mission.outputs.brief`. That mapping was never added. The genesis job's reference to `${{ needs.detect-mission.outputs.brief }}` had been silently resolving to an empty string this entire time. Found while adding the equivalent `instruction` field for the new BUILD chain and checking whether the existing `brief` mapping already existed as a pattern to follow — it didn't.

## Impact
The genesis chain (`run_genesis_chain.py`), built and wired into the workflow as part of `DAM-000001`, had never actually been capable of receiving a real brief in a live run, despite the bug that was supposed to fix exactly this being marked AMENDED and verified. The verification performed for `DAM-000001` tested the script in isolation, not the full cross-job data flow — a real gap in what "verified" meant for that amendment.

## Reuse Guidance
When a fix involves data passed between GitHub Actions jobs (not just within a single job's steps), verification must include checking the job-level `outputs:` block, not just the step that writes to `GITHUB_OUTPUT`. A script-level test that the value is written correctly is necessary but not sufficient — the consuming job's actual reference syntax must also be confirmed to resolve. This is itself a generalisable instance of a pattern worth naming: a fix verified at the wrong layer can look complete and still not work end to end.
