## Finding
PER-000016's formal title, "QA and Governance Director," directly violated STD-000004 Section 4, which lists "QA" as a prohibited term (use: Verification) — in the same document. 21 instances across 16 files.

## Impact
A standard was being violated by a persona name baked into the doctrine that the standard itself governs — an internal contradiction that a manual read would likely never catch, since each individual file looked unremarkable on its own.

## Reuse Guidance
Run a programmatic scan of a standard's own prohibited-terms list against the full doctrine corpus periodically, not just when writing the standard. A new persona, authority, or operation written later can violate an existing standard without anyone noticing, because nobody re-checks old standards against new content by default.
