## Finding
`EMS_DOCTRINE_INVENTORY.md`'s "Other Key Documents" table — the section of the one document whose entire stated purpose is being a complete index — listed 7 of 12 actual root-level markdown files. Missing: `PLATFORM_ONBOARDING_CHECKLIST.md` (being renamed in the same fix), `HOW_TO_FIRE_A_MISSION.md`, `EMS_VOCABULARY_AUDIT.md`, `PLATFORM_AGNOSTIC_AUDIT.md`, `PERSONA_SENSE_CHECK_REPORT.md`.

## Impact
The single document meant to answer "what exists and where" for anyone returning to this EMS was itself incomplete in exactly the category most likely to be consulted first by a new reader — the list of other documents to read.

## Reuse Guidance
A "complete index" document should be checked against the actual filesystem (`ls`) before trusting its own table of contents, the same way artefact ID citations were checked programmatically against reality in DAM-000007. This was caught by manually running `ls *.md` rather than trusting the existing table — the same principle, applied one level higher (a document about documents, not just a document about IDs).
