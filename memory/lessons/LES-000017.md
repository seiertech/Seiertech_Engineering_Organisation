# LES-000017 — HOW_TO_FIRE_A_MISSION.md had gone stale

| Field | Value |

## Finding
`HOW_TO_FIRE_A_MISSION.md` still stated, as of being checked, that Genesis missions had no executor and would only receive a "not yet built" comment — true when the document was written (`DAM-000001` had not yet happened), false by the time it was checked (`DAM-000005` gave Genesis a real v1 executor). The document also never mentioned the BUILD mission format at all, because BUILD did not exist yet when the document was written.

## Impact
A user following this document to fire a Genesis mission would have been told to expect a polite decline, when the actual system would run a real chain. A user wanting to fire a BUILD mission would have found no format documented at all, despite one existing and working.

## Reuse Guidance
Reference documentation describing "what currently works" needs to be checked for staleness specifically at the moment someone is about to rely on it to do something real — not just when it's convenient to update during the commit that changes the underlying behaviour. The actual trigger that caught this was a founder asking for a literal walkthrough, which forced verification against the real parser rather than trusting the document's own prior content.
