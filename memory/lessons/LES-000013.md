# LES-000013 — Documentation Curator personas never referenced OPR-000012

| Field | Value |
|---|---|
| Lesson ID | LES-000013 |
| Category | DOCTRINE_GAP |
| Source | Full left-to-right/right-to-left doctrine sweep |
| Date | 2026-06-30 |
| Doctrine Amendment | DAM-000007 |
| Status | AMENDED |

## Finding
The Documentation and Knowledge Curator persona (both Team 1 PER-000018 and Team 2 T2-PER-000018) — whose entire job description centres on lesson capture and knowledge curation — contained zero references to OPR-000012 Doctrine Amendment anywhere in either file. Cause: both persona files were written before OPR-000012 existed, and were never retrofitted when that operation was built later in the same session.

## Impact
A persona instance reasoning from its own definition file would have no instruction to actually trigger OPR-000012 when a mission surfaces a genuine lesson — despite OPR-000008 (which this persona owns) explicitly pointing at OPR-000012 since DAM-000003. The persona's own operating instructions had silently fallen behind the operation it's supposed to execute.

## Reuse Guidance
When a new Operation, Authority, or Standard is built mid-session, explicitly check which existing personas' job descriptions should reference it, not just the operations/authorities documents that directly govern it. A persona is the most likely place for this kind of staleness to hide, because nothing automatically re-reads persona files when new doctrine is added elsewhere.
