# LES-000007 — Lessons were not load-bearing without a register

## Finding
Every real finding across this session — five solid lessons before this one — existed only in commit messages and one-off audit markdown files. `OPR-000008` Knowledge Capture had always instructed personas to "extract lessons learned," but no conformant `REG-NNNNNN` register existed to receive them, and nothing required a lesson to actually produce a doctrine change.

## Impact
Without this fix, every one of these lessons would have been readable once and then effectively lost — the next session, or the next founder, would have no structural reason to know they existed, let alone to check whether the same bug class had crept back in.

## Reuse Guidance
This is the lesson about the mechanism itself. Going forward: any genuine finding gets a `REG-000010` entry and, unless explicitly justified otherwise, a `DAM-NNNNNN`. `OPR-000012`'s Step 0 session-start gate is the enforcement point — check it before starting new work, every time.
