## Finding
`.github/scripts/call_nim.py` was the original fix for the session's first bug (the JSON-quoting fault). Every subsequent chain script (`run_intake_chain.py`, `run_genesis_chain.py`, `run_build_chain.py`) defines its own local `call_nim` function instead of importing the standalone one — verified identical across all three at the time of this sweep, but structurally nothing prevents future drift, since there is no shared source of truth being imported.

## Impact
None yet — the three copies happen to be identical. The risk is latent: a future fix to the NIM call logic (timeout handling, error formatting, retry behaviour) applied to one script has no mechanism forcing it to propagate to the other two, and nothing would flag the resulting inconsistency short of another manual sweep like this one.

## Reuse Guidance
When a utility function is extracted into its own file specifically to be reused, verify it is actually imported by the things meant to reuse it, not just present in the directory. A file existing alongside duplicated inline copies of its own logic is a sign the extraction never finished.
