## Finding
`issue_parser.py` correctly extracted the `brief` field into its JSON output for GENESIS missions, but never wrote a `brief=` line to `GITHUB_OUTPUT` — the GitHub Actions mechanism for passing values between job steps. Found while wiring `run_genesis_chain.py` into the workflow, before it was ever run for real.

## Impact
Would have silently broken the genesis workflow job on first real use — the job would have received an empty brief with no error.

## Reuse Guidance
When a script's JSON output and its GITHUB_OUTPUT writes are maintained separately (as they were here), check both are kept in sync whenever a new field is added to one — they drifted because the JSON output was written first and GITHUB_OUTPUT was an afterthought added for a subset of fields.
