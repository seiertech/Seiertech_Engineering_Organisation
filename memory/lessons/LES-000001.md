## Finding
`scan_repo.py`'s `ROUTE_HINT_PATTERNS` regex only matched `@app.get()`/`@app.post()` (FastAPI style), despite a code comment claiming "FastAPI/Flask" coverage. Flask's actual decorator, `@app.route('/path', methods=['GET'])`, was never matched. Confirmed by building a fixture with real Flask routes and observing `route_files_with_endpoints` come back empty.

## Impact
Every brownfield intake against a Flask-based platform (a common stack) would have silently under-reported its API surface — the Backend Engineering Lead and Integration Engineer personas would have received incomplete evidence without any error or warning.

## Reuse Guidance
When a code comment claims coverage of multiple frameworks, verify each framework's actual syntax against the regex directly — do not assume shared decorator conventions across frameworks just because they're stylistically similar (FastAPI and Flask both use `@app.X()` but with different method names in the decorator itself).
